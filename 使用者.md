# 捐血報到系統 - 功能說明

## 技術架構

| 項目 | 說明 |
|------|------|
| 前端 | GitHub Pages（靜態網站） |
| 後端 | Google Apps Script（API） |
| 資料庫 | Google Sheet（活頁簿 `blood_donation`） |

---

## Google Sheet 欄位結構

| 欄 | 欄位名稱 | 說明 |
|----|----------|------|
| A | 報名時間 | 系統自動填入，格式 yyyy/mm/dd |
| B | 公司行號 | 下拉選單選擇 |
| C | 姓名 | 使用者填寫 |
| D | 電話 | 10碼手機號碼，強制文字格式（保留開頭0） |
| E | 捐血CC數 | 250cc 或 500cc |
| F | 報到時間 | 報到時系統自動填入 |
| G | 報到編號 | 報到時系統產生，0001～9999，文字格式 |
| H | 報到 | 報到完成回填 Y |
| I | 捐血成功 | 捐血後填寫 Y/N |
| J | 禮品領取 | 捐血成功=Y 時系統自動填入 Y；捐血成功=N 時自動填入「無符合資格」；管理者可手動修改 |

---

## 頁面功能說明

### 首頁（index.html）

- 顯示仕興機械福委會 LOGO（去背圖）
- 兩個功能入口按鈕：
  - 捐血前填寫（`pre-donation.html`）：報名 / 現場報到，取得報到編號
  - 捐血後填寫（`post-donation.html`）：捐血完畢後填寫捐血結果

---

### 一、捐血報名（sign-up.html）

**用途：** 活動前讓民眾預先登記。

**填寫欄位：**

| 欄位 | 說明 |
|------|------|
| 公司行號 | 下拉選單（含「其他」自填） |
| 姓名 | 文字輸入 |
| 手機電話 | 10碼，限制只能輸入數字，格式驗證 09xxxxxxxx |
| 捐血CC數 | 下拉選單：250cc / 500cc |

**系統行為：**
- 報名時間由系統自動抓取（不顯示給使用者）
- 以「姓名 + 電話」檢查是否重複報名，重複則顯示警告
- 報名成功後跳轉至完成頁面

**公司行號下拉選項：**
1. 仕興員工／仕興員工親友
2. 董事長親友/獅子會
3. 總經理(青商會/台灣應達)
4. 銀行
5. 貿易商/報關行總務/教友
6. 供應廠商
7. 週邊學校警義消
8. CCK聯誼廠商
9. 其他（可自填，供社群平台或現場民眾使用）

---

### 二、現場報到－已報名（check-in.html?tab=registered）

**用途：** 已事先報名的使用者現場完成報到。

**填寫欄位：**

| 欄位 | 說明 |
|------|------|
| 姓名 | 文字輸入 |
| 手機電話 | 10碼，限制只能輸入數字 |

**系統行為：**
- 從首頁進入時隱藏頁籤列，直接顯示已報名表單
- 以「姓名 + 電話」比對 Google Sheet 已報名資料
- 查無資料：顯示錯誤提示
- 已報到：顯示警告並顯示原報到編號
- 報到成功：
  - 回填報到時間至 F 欄
  - 產生報到編號（G 欄，4位數字，從 0001 開始遞增）
  - 回填 Y 至 H 欄（報到）
  - 跳轉至完成頁，顯示報到編號與橘黃色出示畫面提示

---

### 三、現場報到－未報名（check-in.html?tab=new）

**用途：** 未事先報名的民眾直接現場報到。

**填寫欄位：**

| 欄位 | 說明 |
|------|------|
| 公司行號 | 下拉選單（含「其他」自填） |
| 姓名 | 文字輸入 |
| 手機電話 | 10碼，限制只能輸入數字 |
| 捐血CC數 | 250cc / 500cc |

**系統行為：**
- 從首頁進入時隱藏頁籤列，直接顯示未報名表單
- 直接新增一筆完整資料至 Google Sheet
- 同時產生報到編號，回填報到時間與報到 Y
- 跳轉至完成頁，顯示報到編號

---

### 四、捐血前填寫（pre-donation.html）

**用途：** 捐血前讓民眾以手機號碼查詢報名資料並完成報到；查無資料者可直接現場報名。

**流程：**
1. 輸入手機號碼查詢
   - **查無資料** → 顯示報名表單，可選擇「僅報名」或「報名並現場報到」
   - **查到資料** → 顯示已報名資料確認畫面

2. 確認資料畫面顯示欄位：公司行號、姓名、手機號碼、捐血CC數
   - 點選「完成報到」→ 完成報到並跳轉出示畫面
   - 點選「修改資料」→ 進入編輯畫面（見下方）
   - 點選「返回重新查詢」→ 回到查詢步驟

**修改資料功能（新增）：**
- 使用者可修改：公司行號、姓名、捐血CC數
- 手機號碼為查詢鍵值，不可變更
- 修改成功後回到確認畫面並顯示「資料已更新」提示
- GAS 呼叫：`update_profile`（以電話定位列，更新 B、C、E 欄）

---

### 五、捐血後填寫（post-donation.html）

**用途：** 捐血完畢後由捐血者自行填寫捐血結果。

**查詢方式（擇一）：**
- 手機號碼（10碼）
- 報到序號（4位數字）

**流程：**
1. 查詢資料，系統顯示：報到序號、公司行號、姓名、手機電話、捐血CC數
   - 查詢結果資料會快取於前端，後續畫面直接重用（確保公司行號與手機電話在所有步驟正確顯示）
2. 依捐血成功狀態分流：
   - **捐血成功 = 空白** → 顯示填寫表單，選擇「成功 / 未成功」後送出
   - **捐血成功 = Y** → 直接跳轉出示畫面（黃底提示出示給工作人員領取禮品）
   - **捐血成功 = N** → 直接跳轉感謝畫面（紅色愛心 ❤️）
3. 送出後：
   - 回寫捐血成功至 Google Sheet I 欄
   - 若捐血成功 = N，自動將 J 欄（禮品領取）寫入「無符合資格」
   - 捐血成功 = Y → 出示畫面（報到序號、公司行號、姓名、手機電話、捐血CC數、捐血成功）
   - 捐血成功 = N → 感謝畫面

**注意：** 以序號查詢時，`lookup_checkin` 需 GAS 重新部署後才會回傳公司行號與電話；前端已改為從查詢時快取資料顯示，不依賴送出後的回傳值。

**信任制度：** 無需驗證，依使用者自行填寫為準。

---

### 六、完成頁（success.html）

依不同流程顯示對應訊息：

| 類型 | 顯示內容 |
|------|----------|
| 報名完成 | 顯示姓名，告知報名成功 |
| 報到完成 | 顯示報到編號 + 橘黃色「報到成功，請出示畫面給服務人員」提示 |

所有頁面皆有「返回首頁」按鈕。

## GAS API 動作一覽

| action | 用途 | 主要回傳欄位 |
|--------|------|-------------|
| `register` | 報名 | status |
| `checkin` | 已報名者報到 | 報到編號、姓名、捐血CC數 |
| `checkin_new` | 未報名者現場報到 | 報到編號、姓名、捐血CC數 |
| `lookup_phone` | 以電話查詢報名資料 | 公司行號、姓名、電話、捐血CC數、報到、報到編號、捐血成功 |
| `lookup_checkin` | 以報到編號查詢 | 公司行號、姓名、電話、報到編號、捐血CC數、捐血成功 |
| `update_donation` | 更新捐血結果 | 公司行號、姓名、電話、報到編號、捐血CC數 |
| `update_profile` | 修改報名資料 | 公司行號、姓名、電話、捐血CC數、報到、報到編號、捐血成功 |
